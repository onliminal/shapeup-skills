# Customer Request Example

This example shows a solution-first customer request getting reframed into a real problem:

`Frame -> Shape -> De-risk -> Pitch`

## Scenario

Enterprise admins keep requesting a "clone permissions from another user" feature. The real pain is onboarding new hires into complex access profiles without rebuilding the same role setup by hand.

## What this example teaches

- how to push past a feature request and find the problem underneath it
- how to protect the appetite from expanding into general RBAC redesign
- how to turn "customers want X" into a narrow, buildable bet
