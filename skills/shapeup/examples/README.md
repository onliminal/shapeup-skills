# Shape Up Examples

These examples are completed sample runs for the Shape Up skill. They are meant to teach:

- conversational tone and pacing
- what "good enough" depth looks like in each phase
- how phase handoffs work in practice
- what the finished artifacts should look like

They are illustrative, not normative. The behavioral source of truth still lives in `references/*.md`.

## Included examples

### New product

Path: `examples/new-product/`

Shows a raw product idea moving through Explore, Evidence, Frame, Shape, De-risk, and Pitch.

### Internal tool

Path: `examples/internal-tool/`

Shows a more direct planning flow for an internal ops tool where the problem is already specific enough to skip Explore and Evidence.

### Customer request

Path: `examples/customer-request/`

Shows how a solution-first customer request gets reframed into a real problem before shaping a feature.

### AI workflow

Path: `examples/ai-workflow/`

Shows an AI/workflow concept that benefits from both Explore and Evidence before it earns appetite.

## Structure

Each example contains:

- `README.md` — what the example is teaching
- `transcript.md` — an abridged sample conversation
- `shaping/{project-slug}/...` — the resulting artifacts from that run

The transcripts are intentionally selective rather than verbatim. They focus on representative turns, checkpoints, and handoffs so the examples stay readable.
